# HonestFlow — структура после рефакторинга

Цель рефакторинга: не менять логику программы, а сделать проект читаемым по папкам и именам классов.

## Карта проекта

- `Application/Bootstrap` — запуск приложения, выбор режима GitHub/local, сборка основных сервисов.
- `Forms` — только WinForms-экраны и обработчики UI.
- `Models` — DTO/модели данных: ИП, версии, ответы API, статус ЛМ.
- `Services/Auth` — авторизация по списку ИП.
- `Services/Core` — сервисы UI-лога и прогресса.
- `Services/Installation` — сценарий установки и проверка версий.
- `Services/Installation/Planning` — план установки: какие компоненты нужны, какие файлы требуются.
- `Services/Lm` — высокоуровневая проверка ЛМ ЧЗ.
- `Services/System` — диагностика системы и управление службой Regime.
- `Infrastructure/Api` — HTTP-клиент ЛМ ЧЗ.
- `Infrastructure/Configuration` — пути приложения, локальные и удалённые конфиги, совместимый фасад `ConfigManager`.
- `Infrastructure/Downloads` — работа с GitHub Release assets и кэшем дистрибутивов.
- `Infrastructure/Installers` — запуск конкретных установщиков: ЛМ, АТОЛ, ЕСМ, контроллер.
- `Infrastructure/Logging` — файловые логи.
- `Infrastructure/Process` — запуск внешних процессов.
- `Infrastructure/Services` — низкоуровневые Windows-сервисы и реестр.
- `Helpers` — вспомогательные функции, которые пока не привязаны к конкретному слою.

## Что принципиально изменено

1. `ConfigManager` больше не является большим бог-классом. Он оставлен как фасад совместимости, а реальная работа вынесена в:
   - `LocalConfigRepository`
   - `RemoteConfigRepository`
   - `ObfuscationService`
   - `AppPaths`

2. Запуск приложения вынесен из `MainForm` в `ApplicationStartupService`.

3. Логи перенесены в `%ProgramData%\HonestFlow\logs`, чтобы логирование не ломалось при запуске из `Program Files` или защищённой папки.

4. Установка теперь строит понятный `InstallationPlan`, а не сразу перемешивает проверку версий, поиск файлов, скачивание и запуск установщиков.

5. Скачивание установщиков теперь проверяется. Если файл не скачался, установка компонента не продолжается как будто всё нормально.

6. Результат установки АТОЛ/ЕСМ/Контроллера теперь учитывается. Успешный лог пишется только если установщик реально вернул успешный результат.

7. Убрано логирование введённого пароля при неудачной авторизации.

8. Убрана рекурсия из обработки конфликта ИНН. Теперь используется цикл повторной проверки.

9. Добавлен `ProcessExecutionResult` и расширенный `ProcessRunner.RunDetailed` для будущей нормальной диагностики внешних процессов.

## Что сознательно не ломал

- Сценарий авторизации оставлен прежним.
- Формат `ips.json` оставлен прежним: XOR 0xAA + Base64.
- GitHub Release-подход оставлен прежним.
- Названия основных публичных методов `ConfigManager`, `Logger`, `ProcessRunner` сохранены, чтобы минимизировать риск поломки.
- WinForms-формы оставлены в namespace `HonestFlow`, чтобы не ломать Designer.
